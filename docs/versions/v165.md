# v165 次期への宿題

v164時点で積み残した課題・検討事項の一覧。

---

## 要対応

### 1. v164 のバージョンドキュメント（structure.json / notes.md）が古い状態

`docs/versions/v164/notes.md` および `structure.json` は、このセッションの修正（アイコン再生成・ListVoices APIキー修正・CLAUDE.md改善）が反映されていない。

- バージョン管理プロトコルに従い、v164ドキュメントの更新またはv165として正式記録すること

### 2. アイコンのデザイン確認

再生成したアイコン（夜空・三日月・「夜」文字）について、実機でのホーム画面追加時の見た目を確認すること。

- iOS Safari / Android Chrome でのホーム画面アイコン表示
- maskable アイコンとしてのセーフゾーン内に主要デザインが収まっているか

### 3. `docs/versions/v164/structure.json` の `sw.js` エントリ

`structure.json` に `sw.js` が `not_implemented` として記載されているが、CLAUDE.mdでは「使用禁止」と定めており、記述の意図が不明確。

- 次バージョンのドキュメント生成時に削除または「対応予定なし」として明記すること

---

## 検討事項

### 4. Google TTS APIキーのテスト

ListVoicesおよびfetchGoogleAudioの両方でヘッダー送信に統一済みだが、実際のAPIキーで動作確認が取れていない。

- APIキー保存・音声一覧取得・試聴・読み上げの一連フローを実機で確認すること

### 5. CLAUDE.md の「バージョン更新」記述との整合

プロトコル手順の「ステップ4」で `<span class="ver">` が明記されたが、アーキテクチャ説明の `APP_VER` 変数説明（「`<title>` と同期」の記述）が3箇所存在することを反映していない点が残る。

---

## 注意事項

- 上記はすべて「確認・検討」レベルであり、緊急のバグではない
- コード修正を行う場合は必ずCLAUDE.mdのルールに従い、ユーザー確認が必要な変更は勝手に進めないこと
